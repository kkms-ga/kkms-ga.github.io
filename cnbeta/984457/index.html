Windows安全应用程序或阻止Windows 10 2004的安装

May 28, 2020

微软已经证实 Windows Security 安全应用程序存在一个 bug，或导致 PC 无法完成 Windows 10 2020 五月更新（Version 2004）的安装。若不在软件中修改 Windows Security 的内存完整性设置，这个兼容性问题就无法自行消除。如果不想放弃内存完整性功能，也可尝试从制造商那里获取新版显示驱动程序，以解除 Windows 10 2004 的升级屏蔽。

访问:

微软中国官方商城 - Windows

如需临时关闭 Windows Security 的内存完整性检查功能，可参考如下操作：

● 前往 Windows 10 设置；

● 移步至 更新和安全；

● 转入 Windows 安全；

● 点击 设备安全性；

● 选择 核心隔离详情；

● 切换 内存完整性设置；

● 重启计算机。

然后可在 Windows Update 中手动检查 Windows 10 2004 更新，或借助更新助手（Update Assistant）获取。

如果仍然看不到更新，请确保设备已更新 Intel / 英伟达的 GPU 显示驱动程序，或者相关厂商的蓝牙设备驱动程序。

微软之前曾警告，若选择忽略安全性问题、并使用媒体创建助手（或其它程序）手动强制更新 Windows 10 2004，可能会引发一系列不可知的后果。

软件巨头或在后续推出永久修复补丁，但目前看来还是稳妥至上。今年 3 月份的时候，该公司还承认了与内存完整性有关的另一个漏洞，其可能引发包括系统故障在内的严重问题。
