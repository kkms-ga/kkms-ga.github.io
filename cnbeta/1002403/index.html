[图]遵循GPT分区规范：Windows 10 20H1开始正确放置恢复分区

July 13, 2020

在用户升级系统新版本的时候，如果 Windows 检测到恢复分区上没有充足的空间，那么它有可能会自动创建一个新的恢复分区。如果未分配的磁盘空间并没有放置在卷分区的后面，那么 Windows 就无法扩展卷分区。

在某些 PC 设备上，WinRe（Windows Recovery）分区作为第一个分区放置，这使得用户或操作系统本身无法扩展卷。在基于 UEFI 的 GPT 存储磁盘上，Windows 设置默认将恢复分区定位在第一个分区时，就会发生这种情况。在某些配置中，恢复分区也会位于 MSR 和 Windows 分区的中间位置。

如果 Windows 10 功能升级需要更多的空间，或者你想要扩展系统驱动器的空间时候，就会出现问题。一旦恢复分区无法被扩展，那么就会通过缩小系统分区来创建新的恢复分区，原来的恢复分区就会被淘汰。您可以删除过时的恢复分区或扩展它，以避免创建一个新的恢复分区。

不幸的是，目前微软官方提供的磁盘管理工具无法做到这点。微软 Contingent 工作人员在论坛帖子中指出，要想对卷分区的位置进行任何修改，需要使用 “diskpart.exe “命令行工具或第三方应用程序。

而在 Windows 10 May 2020（20H1/Version 2004）功能更新中，解决了分区定位的这个根本缺陷。从该功能更新开始，Windows设置现在使用推荐的分区布局进行清洁安装。

此前，当你使用Windows Setup 清理安装 Windows 10 时，卷分区的创建布局如下:

从上面的截图中可以看到，Windows Setup 默认将恢复分区定位在第一位，这违背了微软自己的分区指南。微软指出：“我们建议您将此分区（恢复）紧随 Windows 分区之后。这允许 Windows 以后在未来更新需要更大的恢复映像时修改和重新创建分区。”

而在 Windows 10 Version 2004 功能更新开始，Windows Setup 会在 Windows 分区之后创建 WinRe（恢复）分区。

如上图画面所示，Windows 10 Verison 2004 的 Windows Setup 现在使用了正确的分区布局。旧的布局对一些用户来说是一个巨大的问题，因为它创建了多个恢复分区或阻止用户扩展系统驱动器的空间。在微软的论坛上有几个帖子，用户抱怨多个恢复分区或无法扩展卷空间。随着 Windows 10 Version 2004 的推出，Windows Setup 终于遵循了微软自己的 GPT 分区指南。
