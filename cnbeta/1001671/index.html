Chrome和Firefox浏览器将很快支持全新的AVIF图像格式

July 10, 2020

继流媒体视频网站 Netflix、Windows 10 操作系统、VLC 视频播放器、以及某些图像编辑器之后，Google Chrome 和 Mozilla Firefox 浏览器也终于添加了对全新的 AVIF 图像格式的支持。作为一种新面世的图像格式，AVIF 最大的特点就是轻量和免授权。与现有的 JPEG、PNG、甚至 WebP 等图像格式相比，AVIF 的压缩效率可以做到更高，Netflix 等企业更是对其赞赏有加。

AVIF 又名 AV1 图像文件格式，特点是使用了开源、免版税的 AV1 编码压缩，并以 HEIF 文件格式进行存储。

据悉，AV1 源自谷歌、思科、Xiph.org（也曾与 Mozilla 合作）在 2015 年联手开发的一种视频编解码格式。推出后不久，很快就成为了市面上最受欢迎的免版税视频编解码器之一。

随着 AV1 的成熟，背后的开放媒体联盟决定顺水推舟，打造一种全新的图像文件格式，以借助 AV1 压缩算法来减少 Web 其它应用场景的图像体积。

借助 AV1 的东风，AVIF 图像格式很快收获了许多企业的青睐。比如 2018 下半年的时候，流媒体视频网站 Netflix 就已经提供了对它的支持，甚至早于相关标准在 2019 年 2 月正式批准之前。

不久后，更多流媒体视频网站、以及视频播放器（比如 VLC）和图像编辑器（比如 GIMP）的开发者，也纷纷将 AVIF 图像格式拥入怀中。

值得一提的是，微软在 Windows 10 2019 五月更新（Version 1909）中引入了对 AVIF 的支持，从而极大地推动了对新图像格式的采用。

用户只需从 Microsoft Store 中安装正确的 AV1 视频编解码器（免费下载），即可让 Windows 10 上的画图和文件管理器等应用程序，正确地识别和加载 AVIF 格式的图像。

未来几个月，还将有许多开发者为自家软件添加对 AVIF 图像格式的支持，比如 Web 浏览器领域的 Google Chrome 和 Mozilla Firefox 。

在巨头的带动下，相信许多基于 Chromium 内核的第三方 Web 浏览器也会加速跟进。

据悉，从 2020 年 1 月开始，Mozilla 就已经在努力为 Firefox 浏览器添加对 AVIF 图像格式的支持，最初计划在 5 月份随 Firefox 76 一同发布。

遗憾的是，受新冠病毒引发的 COVID-19 疫病大流行的影响，Mozilla 最终推迟了这项功能更新。如果一切顺利，其有望在 8 月份随 Firefox 80 一同到来。

至于 Google Chrome，其计划在 8 月的某个时候随 Chrome 85 一同发布，而且此后会默认向所有用户启用对 AVIF 图像格式的支持。

Gerrit 在一篇文章中指出，谷歌已经直接跳过了功能测试这个阶段，因其认为在浏览器中添加对 AVIF 图像格式的支持仅存在相当低的风险。
