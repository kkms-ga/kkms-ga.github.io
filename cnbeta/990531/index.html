微软正在修复Chrome浏览器的意外导航行为

June 13, 2020

外媒报道称，微软正在帮助 Google Chrome 修复一个意外的导航行为。比如在将文件拖放到 Twitter 或 Facebook 等网站时，如果你不小心将文件拖放到上传接收框所在的区域之外，就很可能导致浏览器跳转打开新文件，导致当前标签页上的工作被打断。针对此问题，微软表示其已开发出一个修复程序，以改善 Chrome 和 Edge 的意外导航行为。

庆幸的是，微软已经在 Edge Canary 中测试这项改进，此外谷歌也在 Chrome Canary 中展开了测试。

打上这个补丁之后，如果用户不小心将文件拖放到规定区域之外，那它默认将在新建的标签页中被打开，而不是直接替换掉当前标签页上的内容。

此外，如果你在 Windows 上安装了第三方桌面应用程序，那它很可能需要指定不同的文件路径，比如 64 位程序默认的 Program Files 文件夹、以及 32 位程序默认的 Program Files（x86）文件夹。

微软显然不希望用户将 32 / 64 位应用程序混乱地放在一起，以避免可能的兼容性问题。

尴尬的是，即便你安装了 64 位版本，某些基于 Chromium 内核的浏览器（比如 Google Chrome / Edge），它们的默认安装路径竟然还是 C:Program Files（x86）。

Chrome 开发团队称，这么做是为了保持简单，因而没想到更改，且目前并未发现对用户有什么影响。

不过未来的 64 位 Chrome 浏览器，将很快默认安装到 C:Program Files 文件夹中。
