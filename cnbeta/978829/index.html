微软与 Redis Labs 达成合作，以改善Redis的Azure缓存

May 14, 2020

在 RedisConf 2020 Takeaway 上，Redis Labs 宣布与微软达成新的战略协议，以将 Redis Enterprise 作为 Microsoft Azure 缓存上的集成托管服务交付。

访问:

微软中国官方商城 - 首页

目前，这项新服务已可在 Private Preview 中使用，它将通过高级企业级功能（包括 Flash 上的 Redis、modules）为 Azure 客户的关键任务工作负载释放 Redis 的功能，以及在将来用于混合云架构的 active geo-redundant 缓存。

Redis Labs 的首席执行官兼联合创始人 Ofer Bengal表示，“Azure Cache for Redis 的新企业层满足了我们从客户那里听到的对完全托管的云数据存储的需求，该存储可为大型数据集节省大量成本，并且最大程度地简化了 Redis Enterprise 的使用，并从其数据中获得所需的信心和弹性迁移到 Azure 时的层。”

微软开发人员部门副总裁 Julia Liuson则表示，“与 Redis Labs 的集成强调了微软对成功进行云和开源协作的愿景，该协作可满足开发人员的需求并支持开源生态系统以进行持续创新。”“我们相信用于 Redis 的 Azure 缓存的新企业层将帮助公司以规模、高可用性和低延迟交付最关键的应用程序。”

Redis Labs 方面介绍称，Azure Redis Cache 上的新企业产品将提高开发人员的生产力，提供更大的操作弹性，并使公司将数据库从本地基础结构迁移到完全在公共云中运行的公司更加省心。具体来说，客户将受益于：

提高开发人员生产力

使用最新版本的 Redis，包括其本机数据结构、概率数据结构、streams、时间序列和搜索数据模型

在几分钟内部署 Redis 集群，并利用基于 Flash 的 Redis 技术，以基于磁盘的数据存储为代价扩展到TB级的数据集

熟悉的 Redis 缓存和 Redis 界面管理经验，使开发人员可以立即开始使用该服务

与其他 Azure 服务的本机集成以实现无缝开发

更高的弹性，安全性和合规性

企业级 SLA，通过 Redis Enterprise 的 active geo-replication 技术提供额外的正常运行时间

支持网络隔离

简化采购和计费 

客户可以通过基于消费的定价实时扩展所需的 Redis 数量，从而降低总拥有成本

Redis Enterprise 消费已成为客户每月 Azure 账单的一部分

有关集成和新服务的其他见解，可访问 Redis Labs 和 Microsoft 博客。新的企业级 Azure 缓存 Redis 预计将在 2020 年下半年公开发布。有关更多详细信息，请参见此处。
