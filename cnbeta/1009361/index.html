Windows 10 Build 20180发布：邀请更多用户测试磁贴等新功能

July 30, 2020

面向 Dev 频道用户，微软今天发布了 Windows 10 Build 20180 预览版更新。本次更新幅度并不是很大，并没有引入任何新功能，主要是对某些系统特性进行了优化调整。

Windows 10 Build 20180 更新日志

在7月1日发布的Windows 10 Build 20161 版本更新中，微软发布了包括可跟随主题变化磁贴颜色在内的诸多新功能。对此微软表示在今天发布的新版本会面向更多 Dev 频道用户开放这项功能。不过上周 Build 20175 发布将网站固定到任务栏功能目前依然仅向部分 Insider 成员开放。

针对开发者的更新

Windows SDK 现在已经和 Dev 频道同步。每当有新的操作系统构建到 Dev 频道时，相应的 SDK 也会被传送到开发频道。您可以随时从 aka.ms/InsiderSDK 安装最新的 Insider SDK。SDK 版本将和 OS 版本一起在 Flight Hub 存档。

更新和改进

● 基于用户反馈，微软调整了二合一设备的平板电脑姿势逻辑，现在只适用于使用单屏时。

● 基于用户反馈，微软正在更新开始菜单的所有应用列表中的新文件夹图标，使其更小一些，以便更好地与其他图标的大小保持一致。

● 微软更新了“默认应用”设置页面的搜索框，以便于提高性能

修复的内容

● 修复了上个版本中当放大率高于 100% 时，放大镜并没有跟随光标的问题

● 修复了任务管理器中没有显示 UWP 应用程序发布者的名称问题

● 修复了使用 Alt+Tab 切换到浏览器标签时可能导致崩溃的几个问题。

Windows 10 Build 20180 中的已知问题

● 微软正努力修复某些受Easy Anti-Cheat保护的微软商店游戏可能无法启动的问题。

● 微软正研究有关更新过程在尝试安装新版本时长时间卡死的报告。

● 微软正修复调整UWP应用程序的大小后，最小/最大/关闭按钮被卡在其原始位置。如果你移动应用窗口，位置应该会更新。

● 微软正在调查新任务栏体验对某些被固定的网站不起作用的报告

● 有时任务栏中的 “关闭所有窗口 “操作并不能关闭所有打开的标签。

● 固定网站标签页有时候无法预览

● 微软正努力为现有的固定网页启用新的任务栏体验。在此期间，您可以从任务栏中取消固定网站，将其从 edge://apps 页面中移除，然后重新固定网站。

● 被固定的网站不能显示一个域的所有打开的标签。同时，你可以通过固定网站的主页而不是特定的页面来解决这个问题（例如，固定microsoft.com而不是microsoft.com/windows）。

● Alt + Tabbing到浏览器标签有时会将之前活动的浏览器标签也移动到Alt + Tab列表的前面。

●  打开WSL 1发行版可能会导致错误：“由于线程退出或应用程序请求，I/O操作已被中止”。请查看这个Github发行版以获取最新更新。
