[图]推特发布新API v2：向第三方开发者敞开大门

August 13, 2020

面向企业、学者和第三方开发人员，Twitter 今天发布的 API v2 能够让他们更轻松地在平台上进行创建。事实上 Twitter 原定于上月宣布这个新 API，不过当时出现了非常严重的黑客攻击事件，因此 Twitter 被迫决定推迟。

API v2 不仅是初代 API （2012年发布）的首次重大重建，更重要的是 Twitter 尝试搭建桥梁，缓解和第三方开发者的关系。最初，Twitter 更加注重广告业务，因此对于第三方开发者并不是特别友好，会封杀各种非官方应用程序并引导用户使用官方应用程序。

这些做法自然引起了第三方开发商、部分粉丝的不满。为此，在今天发布的 API v2 中为第三方开发人员提供了对其客户端长期缺少的功能的访问，包括“对话贯穿线、推文中的调查结果、在偏好中置顶某些推文，垃圾推文过滤，更强悍的流过滤和搜索查询语言。你还可以访问实时推文流，而不必强迫第三方在提供新Tweet之前等待”。

这意味着在 API v2 发布之后，第三方 Twitter 客户端（例如 Tweetbot 和 Twitterrific）可以开始集成这些功能。目前，API v2 提供了三个级别，今天发布的是免费（free）级别，开发者能够调用的 API 数量会存在一些限制（例如第三方软件 ping Twitter 来获取数据的频率）等等。

此外，下个级别为称之为“高级”（elevated），相对限制就会减少很多，Twitter 并未公布相关的售价。Twitter 表示 80% 的开发人员都能通过免费级别进行访问。
