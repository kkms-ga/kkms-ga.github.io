Android将迎来新的应用内评价API 以缓解当前评价方式的困扰

August 07, 2020

在决定购买或下载体验某款 App 之前，许多人都会先查看一下其他人在应用商店里的评价。但是对于想要给他人提供真实参考的热心用户来说，谷歌 Play 应用商店的评价系统还是有些不太方便。有鉴于此，谷歌正考虑为 Android 平台引入新的“应用内评价”API 。至于新的评价方式能够产生多少积极的影响，仍有待时间去检验。

据悉，当前的应用评价 API，允许开发者通过弹窗的方式让用户去给当前使用的这款 App 发表评价，但其对用户体验的破坏性也是显而易见的。

即便你不想费心去打字，但想要给这款 App 打个星级评分，也会被强制跳转到谷歌 Play 应用商店的界面，结果反而导致许多用户选择无视。

好消息是，尽管新的应用内评价 API 仍会弹出一个窗口，但用户无需为输入评价文字或星级打分而离开当前正在使用的 App 界面。

显然，谷歌希望通过这种即时、快速、方便的新方案，以见谅减少对用户体验的破坏。然而另一方面，也有一些人担心这种易用性可能导致这种应用内评价机制被滥用。

从 Android 开发者博客上的介绍来看，谷歌确实有提醒开发者最好在什么时候向用户提出评价请求。但具体怎么做，还是取决于应用开发者自己的选择。

此外该 API 还具有上限设置，如果用户选择了忽略，理论上可以避免遭受持续的弹窗轰炸。遗憾的是，谷歌似乎并未提供一劳永逸的“不再询问”选项。
