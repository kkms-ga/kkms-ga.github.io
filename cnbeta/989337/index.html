一些用户投诉其PC被强制安装了Windows 10 2004系统更新

June 10, 2020

五月底的时候，微软开始向 Windows 10 用户正式推送 2020 年 5 月更新。在两周的时间里，软件巨头已经分批次地向符合条件的设备用户交付 Version 2004，以便及时发现问题和确保本次更新的可靠性。按照计划，有需要的人们可以通过 Windows Update 手动检查更新。然而有用户抱怨称，他们的系统又一次被强制升级了。

报告指出，个别计算机自动下载并安装了 Windows 10 2020 年 5 月更新，但用户并未执行 Windows Update 手动加查。

AskWoody 的 Woody Leonhard 推测，问题可能与某个 bug 有关，尤其是那些此前已经选择了‘暂停更新’的设备上。

当‘暂停更新’的计时器走到头时，Windows Update 就会自动执行更新检查，结果正巧碰到了 Windows 10 2004 的大版本更新推送。

微软尚未对此事发表评论，但我们也不是第一次见到软件巨头因 Windows 10 强制升级而接到投诉。

当 Windows 10 刚正式推出的时候，许多老用户已经习惯了停留在较早的 Windows 7 / 8.1 平台。即便微软提供了免费升级的选项，许多人仍对此感到厌烦不已。

然而没想到的是，之后发生了多起因为 Windows 7 / 8.1 被强制更新到 Windows 10 的投诉报告，罪魁祸首就是‘获取 Windows 10’这个不请自来的‘升级小助手’。

这款升级工具会在未经用户同意的情况下，于后台悄悄下载 Windows 10 安装文件，然后在重启系统时浪费大量时间去更新部署 Windows 10 。

尽管微软屡次承诺会完善这一机制、以防止将来再次发生设备被强制升级的麻烦，可事到如今，我们竟然又一次在 Windows 10 2020 年 5 月更新上遭遇了历史重演。

如果这真的是一个无心的 bug 导致，想必微软团队会在后续作出补救。至于真相究竟如何，就看微软是否愿意拉下脸来承认“失误”了。
