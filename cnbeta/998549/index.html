Fedora 开发者讨论放弃支持 Legacy BIOS 的可能性

July 03, 2020

Fedora 开发者正在争论结束 Legacy BIOS 对这个 Linux 发行版的引导支持，只支持基于 UEFI 的安装的好处。考虑到 Fedora 33 GRUB 的变更计划，如果他们只是切换到基于 UEFI 的 systemd sd-boot，以及英特尔计划在2020年结束终止支持 Legacy BIOS，并且 UEFI 在 x86_64 系统中已经非常普遍，那么事情会变得更加容易。Fedora 开发者正在讨论现在开始逐步取消支持 Legacy BIOS 是否为好时机。

当然，并不是所有人都同意 Fedora 放弃支持 Legacy BIOS 的想法。一些使用 2012~2013 年时期硬件的用户对可能无法在他们的无 UEFI 硬件上使用新的 Fedora 版本表示失望。

他们还提出另外一个要点，即使大多数关键组件已经支持 UEFI 启动的虚拟机多年，部分虚拟化设置仍然默认使用或高度依赖 Legacy BIOS 引导。

目前尚未提出正式的提案来终止支持 Legacy BIOS 并只支持 UEFI，不过 Fedora 团队已开始对该问题展开积极讨论。

考虑到系统变更的最后期限已过，相信不会在今年晚些时候看到 Fedora 33 放弃支持 Legacy BIOS，但也许在明年或下一年，我们会看到他们放弃支持 Legacy BIOS 的计划，以更好地关注现代硬件，并探索其他可能性，如从 GRUB 转向 SD-BOOT。
