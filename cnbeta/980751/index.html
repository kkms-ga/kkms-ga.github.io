Rust 发行 5 周年

May 19, 2020

Rust 核心团队发布博客庆祝发行五周年，并反思了自 2015 年以来该语言经历的所有重大变化。Rust 开发人员称，“我们重点介绍了几个示例，这些示例可以最好地展示我们已经改进了多少，向用户显示了他们在哪里犯了错误，并重要地帮助他们了解了为什么它不起作用，以及教会了他们如何解决该问题。”

自 2015 年发布 1.0 版以来，Rust 曾在 Stack Overflow 开发者调查中连续四年被评为“最受欢迎的”编程语言。微软也在 2019 年 7 月讨论了对 Rust 的探索，希望将其作为 C、C++ 和其他语言的替代方案，以此来改善应用程序的安全状况。

Theregister 消息称，微软安全响应中心首席安全工程经理加文·托马斯（Gavin Thomas）在去年的一篇博客文章中曾表示，Rust 是能够满足将 .NET C# 等语言的所有内存安全保证与 C ++ 的所有效率结合在一起的，最有前途的更新系统编程语言之一。四个月后，微软 Hyper-V 团队的软件工程师亚当·伯奇（Adam Burch）则发表了有关在微软内部扩大 Rust 的使用的帖子。

不过，相对来说，Rust 还是一个较为年轻的编程语言，因此其在开发人员中也尚未广为人知。JetBrains 去年进行的一项调查就发现，有 97％ 的受访者表示，其使用 Rust 的时间不到一年。今年 2 月， 谷歌公布其新一代开源操作系统 Fuchsia 的编程语言策略时也指出，“Rust 不是一种广泛使用的语言”，并表示，“终端开发者不支持使用 Rust。”

而另一方面，GitHub 最新的 State of the Octoverse 报告则表明，Rust 是 2019 年增长第二快的编程语言（+ 245％）。事实上，Rust 开始出现在越来越多的项目中，例如 Deno 等。

Rust 核心团队在五周年的博客中表示，“如今，数百家科技公司正在使用它。苹果、亚马逊、Dropbox、Facebook、谷歌和微软等一些最大的科技公司也选择在其项目中使用 Rust 来提高其性能、可靠性和生产率。”

3 月份，Dropbox 软件工程师 Sujay Jayakar 庆祝公司在名为Nucleus的文件同步引擎重写中使用Rust 时也写道： “Rust 一直是我们团队的力量倍增器，对 Rust 的押注是我们做出的最佳决定之一。”

除了诸如内存安全性之类的技术优势之外，Rust 吸引人的部分原因还在于，它与以自己的利益至上而闻名的技术公司并没有紧密的联系，像 JavaScript 和 Python 一样，Rust 很大程度上是由社区驱动的。

对于未来，Rust 核心团队憧憬道，“我们迫不及待地想看到 Rust 及其社区将如何继续发展和变化，以及在接下来的十年中，你们所有人将用 Rust 构建什么！”
