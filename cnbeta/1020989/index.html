Visual Studio 2019 v16.8 Preview 2 发布

August 27, 2020

微软发布了 Visual Studio 2019 v16.8 的第二个预览版本。有些存储库中有多个解决方案。现在，当打开这样的存储库时，解决方案资源管理器将显示解决方案列表以供选择。默认情况下，文件夹视图总是位于顶部。这将打开版本库的根文件夹，双击列表中的解决方案即可进入该解决方案。

Git 集成

此外，还可以使用解决方案资源管理器工具栏中的切换视图按钮，回到视图列表，以轻松地在存储库中的解决方案之间移动。

如果存储库中只有一个解决方案，则 Visual Studio 会默认加载解决方案视图。另外，如果存储库中没有解决方案，Visual Studio 默认情况下将打开文件夹视图。可以使用 Tools | Options 中的 Preview Feature 复选框来启用/禁用此功能。

新的 Git 体验：现在，当你成为 Live Share 会话中的 Guest 时，可以以只读模式查看 Git 窗口的内容。

在此版本中，还可以单击以在分支历史之间切换。

.NET 生产力

现在，C# 和 Visual Basic 支持内联参数名称提示，该提示在函数调用中的每个参数之前插入文字、转换文字和对象实例化的修饰符。

另外，现在可以使用新的 Extract Base Class 重构将成员从选定的类提取到新的基类。

另一个功能是代码修复，可以转换 C# 中的 typeof 实例和 Visual Basic 中的 GetType。使用 nameof 而不是类型的名称可以避免在检索对象时涉及的反射。

WPF、UWP 和 Xamarin.Forms 工具

此版本带来了对 XAML Hot Reload 体验的改进。

另外，此版本开始推出对一项高度要求的功能的支持。表单 “changes only” XAML Hot Reload 现在支持 UWP。定位 UWP 时，这将启用 Xamarin.Forms 项目中仅 XAML Hot Reload 支持的新更改。此功能仍在开发中。

还有，XAML 热重载设置已移至“调试>热重载”（Debugging > Hot Reload）。此版本开始将 WPF、UWP 和 Xamarin.Forms 的 XAML Hot Reload 设置合并到一个统一的设置位置。

以及，首次启用了按平台禁用/启用 XAML Hot Reload 支持的功能，这意味着可以为 WPF、UWP 或两者同时禁用这些体验。
