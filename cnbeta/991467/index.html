[图]Windows Insider现启用Dev/Beta/Release Preview新名称

June 16, 2020

从某种角度来说，Windows 10 的成功很大程度上归功于 Windows Insider 项目的良性循环。该项目不仅让用户提前体验各种新功能，也帮助微软构建了完善的测试系统和反馈机制。在成功运行超过五年之后，近日微软宣布对 Windows Insider 项目进行一些调整。

目前，Windows Insider 项目共有 Fast、Slow 和 Release Preview 三个“Ring”（环），用户可以根据自己的偏好进行选择加入。其中 Fast 更新频率最快，基本上每周都能获得改进。待功能逐渐成熟之后就会进入 Slow ，进行进一步的测试。而 Release Preview 往往跟随功能更新发布，在正式发布之前进行更广泛的测试。

在过去几年中，Windows Insider 项目不断根据实际情况进行调整。2019年年中的时候，微软宣布了重大策略调整，Fast Ring 用于 20H1 功能更新，Slow Ring 用于 19H2 功能更新，Release Preview Ring 用于 19H1 功能更新。

而在调整过程中，微软开始意识到这已经超出了“Ring”（通道）的原始概念。为了更契合 Windows 10 当前的更新情况，微软宣布采取基于 Chromium 的 Edge 命名方式，启用“Dev”、“Beta”和“Release Preview”三个新名称，分别对应此前的“Fast”、“Slow”和“Release Preview”通道。

Dev Channel（开发频道）

是技术娴熟群体的理想选择。Dev Channel 的 Insider 成员将会收到开发周期中最早版本，其中并将包含其工程师的最新工作中代码。这些版本将有粗糙的前沿功能并一些不稳定性，可能会阻止关键活动或需要变通办法。

该频道的新功能将不再跟随 Windows 10 功能更新。该渠道中的新功能和操作系统改进将在未来的Windows 10版本中出现，当它们准备好时，可能会作为完整的操作系统构建更新或服务版本交付。这里的Insider 人员反馈将帮助他们的工程师对最新代码进行重大修复和更改。

Beta Channel（测试版频道）

是尝鲜者的理想选择。Beta频道中的Insiders和IT专业人员可以查看即将到来的Windows 10功能，同时还能获得经过微软验证的相对可靠的更新。

作为Beta频道的一部分，Insiders将看到与特定即将发布的版本相关的构建版本，就像之前的20H1一样。你的反馈在这里将特别重要，因为它将帮助他们的工程师确保在主要版本发布之前确定并修复关键问题。

Release Preview Channel（发布预览频道）

Release Preview Channel 的 Insider 和IT专业人员能够审查即将正式发布的 Windows 10 功能，将会获得高质量更新和某些关键功能。这些版本都将由微软提供支持。

Release Preview Channel 是微软推荐企业在内部广泛部署之前的预览和验证版本。随着微软未来构建和发布Windows 10的方式不断发展，微软可能会推出新的渠道，为Insiders提供新的体验。

和 Office 保持一致

微软非常高兴地宣布频道的命名能够和 Office 保持同步。Insider 成员能够根据自己的需求，在 Windows 和 Office 之间获得相同的体验。Insider仍然需要分别选择加入每个Insider Program，但微软正在寻求随着时间的推移改善这种体验。

下一步

对于 Windows Insiders 来说，移动到 Channel 将非常容易。如果你的设备符合相关的条件，微软会根据你当前的 Ring 自动将你的设备移动到最符合该体验的新频道。你不需要采取任何行动就能实现这一改变。

Insiders 也可以通过 “设置">"更新与安全">"Windows内部人员计划 “来调整这些设置。IT专业人员可以使用其典型的Windows Update for Business或Windows Server Update Service管理工具，或通过使用Azure Marketplace虚拟机(VM)来管理Beta和Release Preview渠道构建。
