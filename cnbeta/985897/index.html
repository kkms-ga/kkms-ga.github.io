Microchip推出Switchtec PAX PCIe 4.0交换芯片

June 02, 2020

为增加 PCIe 连接的扩展性，或者跨多个主机通讯，行业内对于 PCIe 交换 / 桥接芯片的需求也相当旺盛。随着 PCIe 4.0 的兴起，显卡、SSD、FPGA 也陆续添加了对新标准的支持。有鉴于此，Microchip 特地升级了自家的 PCIe 3.0 交换芯片至最新的 4.0 版本。它就是提供多达 100 条通道、支持 52 个设备、以及具有 174 GBps 交换能力的 Switchtec PAX PCIe 交换机系列。

起初，我们曾见过 nf200 这样的设备出现在 EVGA SR2 等高端主板上，然后 Z77 主板上的 PLX PEX 桥接芯片又允许 16 通道 CPU 带来 32 通道的连接性。

同期某些供应商提供了较为落伍的双交换 / 多达 22 个 SATA 端口的解决方案，以及四通道 SLI 交火 / 附加 LSI Raid 控制器。

不过在近期的消费级产品上，我们尚未见到过此类大型 PCIe 桥接芯片的大量使用，主要原因有两个。

首先，PLX 于 2014 年被 Avago 收购（交易估值 3 亿美元），导致芯片价格一夜之间保障三倍。

其次，与消费产品领域的 PEX 8000 相比，下一代 PEX 9000 桥接芯片具有更丰富的特性，支持交换结构之间的连接和故障转移。

随后 Avago 又与博通（Broadcom）合并，但情况维持不变，因为桥接芯片的市场，主要还是集中在服务器领域。

好消息是，随着竞争的转移，Microchip 还是展露头角。在 2019 年的超算大会上，外媒 AnandTech 曾询问有关 PCIe 4.0 交换芯片的事宜，当时该公司透露 Switchtec PAX 产品线将很快到来。

Microchip 提供了 6 种产品，支持范围从 28~100 条通道（最低可拆分到 x1）。这些产品能够全速工作，同时保持完整上下兼容性。尽管有潜在的瓶颈，但客户仍可轻松实现 1~99 的转换。

此外新交换芯片支持每个端口的热插拔，实现低功率 Serdes 连接的可操作性，支持 OCuLink，且可与无源、Host 或光缆一同使用。

通过端点系统和管理软件，客户能够实时访问诊断信号。所有连接都可部分容错和支持旁路，以及复位部分功能，使得跨主机和设备的结构变得非常简单，且支持各种拓补方案。

价格方面，Microchip 表示最终取决于具体的数量和场景。可以想象戴尔（DELL）或超微（Supermicro）这样的行业巨头能够尽快研究出基于该公司 PCIe 4.0 交换方案的新产品（分布式 / 存储设备等）。

普通消费者亦有望在超微的 Z490 主板新品上见到 Microchip 的 PCIe 4.0 桥接芯片，其通过 PEX 8747 芯片将 Intel Comet Lake 上的 PCIe 从 16 扩展到了 32 通道。
