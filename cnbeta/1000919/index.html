谷歌向符合条件的设备用户推送Android 11 Beta 2

July 09, 2020

上个月，谷歌向符合条件的设备用户推送了 Android 11 的首个 Beta 测试版本。现在，随着 Android 11 抵达“平台稳定”（Platform Stability）的开发阶段，谷歌又向符合条件的设备用户推送了第二个 Beta 测试版本。据悉，Android 11 Beta 2 为开发人员引入了一些新的功能和改进。

（来自：Google Blog）

由更新日志可知，Android 11 Beta 2 主要迎来了以下变化。包括（1）最小化行为更改的影响、（2）更轻松的测试和调试、（3）对非 SDK 接口的限制条件、（4）动态资源加载器、以及（5）抵达平台稳定性的一个新的里程碑。

● 为最小化平台行为变更的影响，Android 11 允许有选择性地加入，直到开发者在自己的 App 中设置了 targetSkdVersion 。

● 为帮助开发者测试兼容性，Android 11 已允许通过开发人员选项或 ADB 来强制启用或禁用特定的某项改动。

● 为鼓励开发者逐渐远离非 SDK 的应用程序接口，Android 11 更新了针对非 SDK 接口的限制条件，同时会继续支持公共 API 请求和听取大家的反馈。

（来自：Google Developers）

● 作为远离非 SDK 接口的努力的一部分，开发者希望 Android 11 提供一个可在运行时（runtime）中加载相关资源的公共 API，于是我们在 Beta 2 中迎来了一个新的资源加载框架。

● 为让开发者更好地了解最终变动的发布过程，新的平台稳定性里程碑不仅包含了最终版的 SDK / NDK API，还包含了可能影响大量应用的内部 API 和系统行为。

如果你拥有一部 Pixel 2 / 3 / 3a / 4 智能机，可先申请注册，以便通过 OTA 的方式获取 Android 11 Beta 2 测试版更新。

即便没有 Pixel 设备，亦可尝试借助 Android Flash Tool 刷机工具写入镜像、或在 Android Studio 中使用 Android Emulator 模拟器来体验。
