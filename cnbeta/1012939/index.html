Firefox 79稳定版导致部分Mac用户陷入不明卡顿

August 07, 2020

Firefox 79 是 Mozilla 目前提供的最新稳定版本，包含了功能和安全性方面的全面保障。在 Windows 平台上，Firefox 79 的 WebRender 可充分发挥 Intel 和 AMD 显卡的性能。然而在 macOS 上，Firefox 79 的改进不仅不明显（主要集中在底层），甚至还引入了一个奇怪的 bug 。

Softpedia 编辑 Bogdan Popa 援引某位同事的话称，在更新到 Firefox 79 之后，他感觉浏览器的响应变得非常缓慢，偶尔还会卡住，然后又恢复“正常”的工作状态。

更让 macOS 用户感到难受的是，Firefox 79 似乎会“定期”发生卡顿，但这种间歇性的故障让我们很难找到罪魁祸首。即便采取了取下操作，该问题还是让人感到束手无策：

● 逐个删除 Firefox 79 的附加组件，以排查引发故障的确切原因；

● 清除所有浏览器数据；

● 重新安装 Firefox 79；

● 多次尝试重启设备。

诡异的是，在同一设备上运行的所有其它浏览器，似乎都一切正常。排查到现在，所有矛头都指向了 macOS 版 Firefox 79 本身。

尽管目前该问题引发的关注还不是很广泛，但我们确实在 Reddit 上也看到了类似的 MacBook 使用卡顿报告。

一位网友称：Firefox 一直在莫名其妙地卡断，完全不知道是什么原因。无论我是在电子邮件、社交媒体、甚至搜索引擎中键入一些内容，浏览器都会卡顿个 5~10 秒。

另一位网友称：卡顿的时候，浏览器没能对任何输入操作作出响应。但在恢复过后，它又会在移动鼠标时胡乱响应点击。即便我没用它来执行任何大型任务，也几乎会在尝试某些事情时遭遇冻结。

作为应对，某些网友建议可在安全模式下运行 Firefox 79，只是这么做会让浏览器的整体速度都变慢，就像是互联网连接出现了问题。

现在我们只能寄希望于 Mozilla 在新版中彻底修复困扰 macOS 用户的这个 bug（预计 8 月 25 日发布），等不了的用户，不妨想想其它办法（回滚至旧版、或切换到其它浏览器）。
