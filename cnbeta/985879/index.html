微软回应AppGet创造者剽窃说：肯定其对Windows软件包管理器的贡献

June 02, 2020

据外媒报道，上周，AppGet创造者Keivan Beigi写了一篇博客文章中喊话微软–称Windows Package Manager（Windows软件包管理器）几乎剽窃了他的东西。根据他的说法，作为人才收购，微软联系了他并让他接受了一整天的面试，然后沉默了六个月。

显然，在那天的面试中，他对AppGet的计划包括什么可行什么不可行都被讨论到了。

微软在周末对此做出了回应并给予了Beigi许多Windows Package Manager功能的赞誉，比如安装过程中没有脚本、GitHub中丰富的清单定义、支持所有类型的Windows应用安装程序以及应用程序的无缝更新。然而这看起来不是一个态度明确的道歉，而是承认自己“在Keivan和AppGet上没有做到这一点”。

回复的作者Andrew Clinick也承认，去年夏天他确实见过Beigi，并听取了他关于Windows package manager应该是什么样子的见解。显然，最终的产品是受到了那次谈话的启发。

AppGet是一个开源项目，因此，如果微软将使用或复制一些代码还属于权利范围内。当然，虽然没有法律要求这样做，但把功劳归于AppGet和Beigi是正确的事情，所以公司正在努力使其正确。
