[图]Google Messages即将为RCS信息启用端到端加密

May 24, 2020

根据APKMirror最新获取的Google Messages 6.2内部“dogfood”构建版本，通过深入发掘代码后发现了一项新特性：RCS信息即将实现端到端加密。其中“dogfood”是俚语“吃自己的狗粮”，通常描述软件公司使用自己生产的产品。

最近几年来，人们一直对RCS信息服务充满兴趣，认为它将会是短信、彩信的真正继承者，并且会是苹果iMessage的公开竞争者。而相比较iMessage，RCS一直缺席的一项东西就是端到端加密。

在Google Messages 6.2内部“dogfood”构建版本中，外媒通过发掘代码发现允许用户发送端到端信息加密。事实上，该应用中总共有12个新字符串提到了加密（有时简称为 “e2ee”）。

End-to-End Encrypted Rich Communication Service message

Send end-to-end encrypted message

Chatting end-to-end encrypted with %s

End-to-end encrypted message

目前还没有RCS端到端加密的更多细节。有可能需要双方都使用Google Messages应用才能启用。不过可以肯定的是，发送者和接收者都需要同时拥有良好的网络连接，才能通过这些端到端的加密RCS消息。

如果你们中的任何一个人的连接不好，Google Messages会提供通过短信或彩信作为备用方式发送信息。但在以这种方式发送之前，应用程序会提醒你，短信和彩信是不加密的，需要征得你的同意。
