[图]微软承认Windows Sandbox存在启动错误 重启设备可解决

July 24, 2020

Windows Sandbox 是随 Windows 10 Version 1903 功能更新引入的一项功能，允许用户运行独立的操作系统实例，从而可以安全地测试应用程序和服务。它就像是 Windows 10 系统自带的虚拟机，系统独立在沙盒中运行，所有的系统文件都受到保护，而且当应用程序关闭时，所有的更改都会自动重置。

不得不说，Windows Sandbox 解决了很多用户对安全测试环境的需求，减少了部署 VirtualBox 等应用的必要性。不过微软近日承认，近期发布的累积更新中导致 Windows Sandbox 和 Windows Defender Application Guard（WDAG）出现启动错误。

值得注意的是，目前受影响的功能更新版本包括

Windows 10 Pro, version 1903

Windows 10 Enterprise, version 1903

Windows 10 Pro, version 1909

Windows 10 Enterprise, version 1909

Windows 10 Pro, version 2004

Windows 10 Enterprise, version 2004

包括 Windows 10 May 2020（20H1/Version 2004）在内的功能更新都存在这个错误。在一份微软官方支持文档中显示用户在运行 Windows Sandbox 之后可能会遇到“ERROR_VSMB_SAVED_STATE_FILE_NOT_FOUND (0xC0370400)”或者“E_PATHNOTFOUND (0x80070003)”的错误提醒。

微软表示，目前的临时解决方案非常简单，那就是重启电脑。微软表示：“如果收到上述任意一条错误通知，为缓解这个问题，你需要重启设备”。微软表示目前已经在研发相应的修复补丁，但目前并没有提供具体的上线时间。
