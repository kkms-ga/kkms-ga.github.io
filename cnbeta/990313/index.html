[图]终于要解决了：64位Chrome可安装到C:\Program Files路径下

June 12, 2020

谷歌终于开始着手解决 Chrome 浏览器上的一个问题，或许这和新功能无关，但对于 Windows 用户来说依然是个大新闻。现在如果你在 Windows 上安装 64 位 Chrome，那么应用会在 x86 版的应用程序文件夹存储某些浏览器数据。

例如在 Windows 10 上运行 64 位的 Chrome 浏览器，那么该应用的默认安装路径是“C:Program Files (x86)GoogleChrome”。所以尽管是 64 位版本，但依然会将文件放在 x86 文件夹中。而且这个问题早在 2014 年的时候就在 Chromium Bugs 跟踪器上报道过，但一直没有得到妥善的解决。

在 6 年的等待之后，谷歌终于开始着手解决这个问题。援引外媒 TechDows 报道，最新 64 位 Chrome 浏览器开始安装在正确的文件夹中，这意味着部分文件将会存储在“C:Program FilesGoogleChrome”路径下。

然而，看起来只有新安装的浏览器才会进入这个路径，所以如果你已经在运行Chrome浏览器，你必须删除它，然后重新安装浏览器，将所有文件移到正确的文件夹中。目前尚不清楚谷歌何时正式上线该功能
