Chrome 无障碍访问添新功能：自动生成带标签的 PDF

July 30, 2020

Chrome 无障碍访问技术负责人 Dominic Mazzoni 今日发文宣布，从 Chrome 85 开始，当用户选择打印网页并另存为 PDF 时，Chrome 将自动生成带标签的 PDF。“带标签的 PDF” 即包含有关文档额外元数据的 PDF，包括标题、列表、表格、段落和图像描述一类的内容。

对于残障或是不便阅读网页内容的人士（例如需要使用屏幕阅读器的弱视力者）来说，这项功能有助于提高访问的便利性。除此之外，带标签的 PDF 也可以有其他用途，比如能够使某些软件更轻松地从 PDF 提取数据、自动处理。

标签描述功能在网页中一直可用，Chrome 团队认为是时候将这些元数据也添加到 PDF 中：“我们希望这有助于从 Chrome 浏览器导出的内容可供更多用户访问。”

Chrome 团队发现，许多适合内容创作的软件当前还不支持直接生成带标签的 PDF，通常需要第三方服务来兼容这一功能。现在，通过将其内置在 Chrome 中，他们希望将 HTML 作为文档工作流的人可以利用该功能，轻松生成更兼容的 PDF。

这项工作已进行了两年，目前还处于实验阶段，在 chrome://flags/#export-tagged-pdf 中启用即可开始尝试。接下来的工作重点包括提高生成的 PDF 质量，还包括对 Chrome 内置 PDF 阅读器的改进。稳定版预计八月下旬随 Chrome 85 一同发布。
