清盘清理可能破坏某些设备上的Windows 10 2004系统

July 17, 2020

为了避免大规模推送时遭遇 bug 的集中爆发，微软为 Windows 10 2020 五月更新（Version 2004）采取了分阶段推送的政策。一旦发现某些严重的问题，软件巨头可及时屏蔽向某些类似配置的设备的更新部署，然后在修补程序准备就绪后恢复推送。某些情况下，微软还可通过系统自带的疑难解答工具来修复问题，以避免频繁地介入更新屏蔽。

最近的一个例子，就与 Windows 10 2004 的磁盘清理工具有关。遭遇严重问题的话，甚至可能导致设备损坏到无法启动。

遗憾的是，微软并未分享更多细节，仅表示“在安装 Windows 10 Build 19041.21 更新之后，运行磁盘清理可能导致某些设备无法启动”。

从技术上来讲，安装完 Windows 10 2020 五月更新和面向 Insider 测试者的 19041.21 内部编译版本后，磁盘清理通常会自动启动，结果就是导致某些设备无法再次启动。

有人猜测这个 bug 可能会错误地删除或破坏一些重要的 Windows 10 系统文件，但目前尚未得到官方的证实。

不过微软并未通过升级的方式来解决这个问题，而是提供了专门针对磁盘清理工具的疑难解答程序，目前已可在 Windows 10 设备上启用并自动启动。

本质上，疑难解答程序所做的，只是简单地禁用磁盘清理工具的自动启动，以防止上文推测的意外文件删除。直到设备安装了 19041.84 更新，才会再次解禁。

有趣的是，该疑难解答程序无法手动启动。微软仅向用户提供了一个专用的屏幕界面，来确定该疑难解答程序是否已启动，并且只会在上述问题影响设备时自动运行。

该疑难解答程序将自动运行两次，首先是在 19041.21 编译版本的所有设备上运行，然后在升级到 19041.84 编译版本后再次运行。

如不确定是否已执行，可检查系统中的故障排查历史记录，方法是在“开始 -> 设置 -> 更新与安全 -> 故障排除 -> 查看历史记录”。

当前 Windows 10 2004 仍仅在有限的一批设备上部署，预计今夏晚些时候会迎来更加全面的推送。等不及的朋友，亦可通过媒体创建工具来获取新版安装镜像。
