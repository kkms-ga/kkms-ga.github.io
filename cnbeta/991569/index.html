[图]微软正改进Windows Core OS对Chromium的支持

June 16, 2020

微软正积极推进 Windows Core OS（WCOS）的模块化系统开发。援引外媒报道，在微软 Edge 工程师提交至 Chromium Gerrit 的一个 Commit 中就引用了该系统。根据提交的内容，看起来微软正在优化Chromium，通过修复 “AppContainer “故障，使其在Windows Core OS上更好地工作。

在内容中写道：“在Windows Core OS系统上，应用容器名称的字符限制是50个字符。Chromium目前提供的应用容器名称大于50个字符。这一改变缩短了沙盒进程名称。”AppContainer用于为Win32应用程序提供安全的沙箱环境，它允许应用程序访问专门授予它的资源。

根据该提交的参考资料，该技术巨头似乎也在优化 Chromium，使其在 Windows Core OS 和 Windows 10X 上的Win32容器或沙箱环境内运行，确保不会被黑客利用执行恶意操作。
