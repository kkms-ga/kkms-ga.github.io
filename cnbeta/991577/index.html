[图]升级Windows 10 Version 2004后Chrome出现要求重复登录情况

June 16, 2020

在升级 Windows 10 May 2020（20H1/Version 2004）功能更新之后，部分 Chrome 用户反馈称谷歌会不断登出用户账号，并强迫用户再次登录。一旦你关闭Chrome浏览器，然后再次打开它，这个问题就会再次出现–你必须再次登录。卸载然后重新安装浏览器并不能解决这个问题。

事实上，这个问题在 Windows 10 Version 2004 的预览版中就已经存在，不过由于 Chromium 团队难以重现这个问题，他们原本并不打算进一步调查。幸运的是，谷歌安全研究员 Tavis Ormandy 也遭遇了同样的问题，这促使团队进一步调查这个问题。

这个问题似乎是 Windows Data Protection API (DPAPI) 导致的，该 API 存在于 Windows 2000 以及后续版本，能够帮助保护包括私钥、存储的凭证和其他机密信息数据。

目前尚不清楚导致问题出现的具体原因，不过 Tavis 表示这个问题可能会蔓延到同样基于 Chromium 的 Edge 浏览器上。

在谷歌修复这个问题之前，Tavis推荐临时解决方案如下

1.关闭Chrome.exe进程。通过打开任务管理器(Ctrl + Shift + Esc)，确保它已完全关闭。

2.按住Windows + L锁定系统。

3.输入密码或用Windows Hello配置的验证系统，即可解锁系统。

4.重新打开Chrome浏览器
