[图]Windows 10 v2004可能会触发ESENT警告 导致系统卡顿

July 17, 2020

在升级 Windows 10 May 2020（20H1/Version 2004）功能更新之后，部分用户反馈称收到了来自 ESENT 文件的警告。自 Windows 2000 系统以来的所有版本，都存在可扩展存储引擎（Extensible Storage Engine，缩写ESE）中 ESENT.DLL 文件，被 Windows Update 等诸多 Windows 组件广泛使用。

根据微软版主介绍，在 Windows 10 系统中，ESENT (ESE Runtime)能够在用户登录之后帮助执行桌面搜索、索引多媒体目录等等。在微软 Answer 论坛和 Reddit 社区上，有用户表示升级到新版本之后收到关于 ESENT 的警告。

根据用户的反馈和外媒 Windows Latest 的测试，在事件查看器中可以复现这个 ESENT 642 警告。在事件查看器中的警告如下

Video.UI (23680,D,2) {B8A5865B-DCFF-4019-AA40-BEE2E42C0672}: 由于目前的数据库格式为 1568.20.0，因此无法使用数据库格式功能版本9080 (0x2378)，由参数 0x410022D8 (8920 | JET_efvAllowHigherPersistedFormat)控制。

通过查看事件查看器中的日志可以确认这个问题，目前还不清楚这个问题的根本原因。一位用户指出：“我在玩 R6S 时也像疯了一样卡顿”。另一位用户反馈到：“我从 150fps 变成了 40/50fps，这也导致操作系统卡顿，我玩的时候出现了这个错误，我的游戏开始像疯了一样卡顿了几秒钟。”
