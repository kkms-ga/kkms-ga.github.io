[图]微软提议改善Chrome在Windows 10上性能表现：精简冗余代码

July 05, 2020

在 Edge 浏览器拥抱 Chromium 平台之后，微软加大了对该平台的贡献力度，并携手谷歌修复和解决了耗电、无障碍工具以及开发者工具在内的诸多问题。现在，微软已经着手改善 Chrome 浏览器在 Windows 10 系统上的性能表现了。

根据最新的 BUG 追踪帖子，微软指出谷歌 Chrome 浏览器会运行一段代码来确认 Chrome.exe 浏览器是否被固定在任务栏上。这段代码检查会在 Chrome 浏览器启动 45 秒之后进行，会消耗 300ms 的 CPU。

这个消耗对于大部分配置来说都不明显，不过微软的 Eric Lawrence 认为，修复小问题最终可以提高浏览器的性能。微软表示：“在这项改变之前，需要创建单独的实用程序，需要枚举快捷方式文件夹中的快捷方式，并枚举快捷方式上的所有 shell 动词，以查看是否包括 Unpin from taskbar。”

微软还补充说，这个额外的步骤是不必要的，因为只要在任务栏目录下有快捷方式就应该足够了。微软的修复将删除枚举shell动词的代码，并在浏览器进程中运行代码，而不是单独的实用程序进程。
