微软希望在Windows 10/10X上为Chromium带来完整的游戏手柄API支持

July 12, 2020

随着越来越多的游戏通过浏览器流媒体在线进行，浏览器中对专用游戏控制器的支持需求在增加。Chrome浏览器已经有了Gamepad JavaScript API，但微软正在寻求增加对Windows.Gaming.Input API的全面支持作为替代后端，以便对控制器上的专用硬件提供更大的支持，比如Xbox One游戏手柄上的触发器，以及Elite控制器上的拨片。

因此，微软在Chromium开发者论坛上提出了一个功能请求，要求更新Chromium中的Gamepad Javascript API，增加对Windows.Gaming.Input(WGI)的支持，作为XInput之外的另一个后端，此举被认为将支持Windows 10上更广泛的现代游戏手柄设备。

他们还指出，像Raw Input这样的低级API今后将不会在WCOS设备上得到支持，而Windows.Gaming.Input API将在Windows桌面（Win32应用）和Windows 10X上工作。这一改变还将带来其他优势，比如提供对P&P（即插即用）通知的支持。

微软已经开始了添加API的工作，当然，这些改动是否会被并入Chromium代码库还有待观察，一旦成功将影响深远，因为这将最终为Chrome、Edge、Opera以及更多基于Chromium的浏览器带来全面的原生Windows游戏手柄支持。
