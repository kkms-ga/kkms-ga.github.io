[图]谷歌成立Open Usage Commons 帮助开源项目管理商标

July 09, 2020

联合咨询合作伙伴 SADA Systems 以及诸多学术领袖，今天谷歌宣布成立 Open Usage Commons，旨在帮助开源项目管理其商标。谷歌开源总监 Chris DiBona 表示：“过去几年间，商标问题日益凸显。通常情况下，开源软件中并不会涉及太多商标。不过在谷歌，以及那些更高层次、更受欢迎的开源项目中使用的商标变得越来越多。”

而现有的开源许可比较注重版权和专利，商标往往被很多人忽略掉。即便是 Apache 许可，也基本上会撇开关系。传统上，开源许可证并不涵盖商标，因为生态系统中根本没有多少商标需要担心。这里的一个例外是Linux，这个商标现在由Linux Mark Institute代表Linus Torvalds管理。

DiBona 解释说：“我们想要做的是围绕着如何以分享专利和版权的相同方式在开源许可[……]中分享商标提供指导，而这个想法基本上是提供这种指导，你知道，提供商标文件，如果你愿意的话，可以包括在你的源代码中。”

谷歌自己正在将自己的三个开源商标放入这个新的组织中：移动端的 Angular Web 应用框架、Gerrit 代码审查工具和 Istio 服务网格。“它们三个都有点适合做这种实验，因为它们正在谷歌进行积极的开发，它们有与之相关的商标，它们有标识，在某些情况下，还有吉祥物。”

其中一个吉祥物是 Diffi，全称是 Kung Fu Code Review Cuckoo。正如DiBona所指出的那样，“我们正在努力想出字面上最糟糕的吉祥物。” 现在，该商标由开放使用共享组织管理。
