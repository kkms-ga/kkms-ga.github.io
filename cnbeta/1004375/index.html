[图]Edge 84稳定版发布：优化集锦 默认禁用TLS 1.0/1.1

July 17, 2020

时隔 6 周时间，Edge 浏览器的最新稳定版 v84.0.522.40 正式发布。新版本为 IE 模式改善了站点列表下载时间，在“以管理员身份运行”运行时允许用户登录浏览器等等。

下载地址:https://www.microsoftedgeinsider.com/zh-cn/download

主要改进内容如下

● 该版本 Microsoft Edge 改善了 Internet Explorer 模式下的站点列表下载时间。

在没有缓存站点列表的情况下，微软将 Internet Explorer 模式站点列表的下载延迟降低到 0 秒（从 60 秒的等待时间降低）。微软还添加了组策略支持，以应对 Internet Explorer 模式主页导航需要延迟至下载站点列表的情况。如需了解更多信息，请参阅 DelayNavigationsForInitialSiteListDownload 策略。

● Microsoft Edge 现在允许用户在 Windows 10 上 “以管理员身份运行 “时登录浏览器。

这将帮助客户在 Windows 服务器上或远程桌面和沙箱场景下运行 Microsoft Edge。

● Microsoft Edge 现在在全屏模式下提供全鼠标支持。

现在无需退出全屏模式，您可以使用鼠标访问标签、地址栏和其他项目。

● 改善网购。

为保存的借记卡或信用卡添加自定义昵称。现在，您可以在进行在线购买时区分您的信用卡。命名您的借记卡或信用卡可以让您在使用自动填写选择支付方式时选择正确的卡。

● 默认禁用 TLS/1.0 和 TLS/1.1。

为了帮助发现受影响的网站，您可以设置 edge://flags/#display-legacy-tls-warnings 这个实验 Flag，以使 Microsoft Edge 在加载需要传统 TLS 协议的页面时显示一个非阻塞的 “不安全"通知。SSLVersionMin 策略允许重新启用 TLS/1.0 和 TLS/1.1。这个策略至少在 Microsoft Edge 88版之前都是可用的。如需了解更多信息，请参见 Microsoft Edge 即将推出的影响网站兼容性的更改。

● 集锦改进

  ○ 新增注释功能

让你可以在一个集锦中的项目上添加注释或评论。注释允许被分组，即使您对集锦中的项目进行排序，也会保持与项目的关联。要尝试这个新功能，请右击一个项目并选择 “添加备注”。

  ○ 更改集锦中笔记的背景颜色

你可以使用颜色编码来帮助你组织信息，提高工作效率。

  ○ 改善性能

在性能上有明显的提升，与之前版本的 Microsoft Edge 相比，可以让你在更短的时间内将集锦导出到Excel。

● 额外的 Microsoft Edge API 支持

  ○ The Storage Access API

当用户提供直接意图允许存储时，该 API 允许在第三方上下文中访问第一方存储，否则将被浏览器的当前配置所阻止。

随着隐私对用户越来越重要，要求更严格的浏览器默认设置和用户选择设置，如阻止所有第三方存储访问，越来越常见。

虽然这些设置有助于提高隐私性，并阻止未知或不信任的方的不必要的访问，但它们可能会产生不必要的副作用，例如阻止访问用户可能想要查看的内容（例如，社交媒体和嵌入式媒体内容）。

  ○ The Native File System API

这意味着你可以通过原生文件系统API给予网站编辑文件或文件夹的权限。

● 对 PDF 文件的改进

  ○ Read Aloud for PDF

可以让用户在执行其他可能对他们很重要的任务时收听PDF内容。它还可以帮助视听学习者专注于阅读内容，使学习更容易。

  ○ PDF文件编辑得到改进。

现在，您可以将对PDF进行的编辑保存到文件中，而不是每次编辑PDF时都保存一份副本。

● 翻译功能

Microsoft Edge 现在可以在沉浸式阅读器中启用翻译功能。当用户打开沉浸式阅读器视图时，他们可以选择将页面翻译成他们所需的语言。

● 开发者工具

DevTools 支持自定义键盘快捷键以匹配您的编辑器/IDE，其中包括 VS Code。

隐私策略调整

微软新增了 7 项新的策略，从 Microsoft Edge Enterprise 登录页下载更新的管理模板。添加了以下新策略：

● AppCacheForceEnabled

允许重新启用AppCache功能，即使默认情况下已关闭。

● ApplicationGuardContainerProxy

配置 Application Guard Container Proxy 的设置。

● DelayNavigationsForInitialSiteListDownload

要求在标签导航之前，企业模式站点列表可用。

● WinHttpProxyResolverEnabled

使用 Windows 代理解析器。

● InternetExplorerIntegrationEnhancedHangDetection

为Internet Explorer模式配置增强的挂起检测。

● NativeWindowOcclusionEnabled

启用隐藏本地窗口。

● NavigationDelayForInitialSiteListDownloadTimeout

设置企业模式站点Li的标签导航延迟时间。

废弃的策略

● AllowSyncXHRInPageDismissal

允许页面在解散时发送同步的XHR请求。

● BuiltinCertificateVerifierEnabled

确定是否使用内置证书验证器来验证服务器证书。

● StricterMixedContentTreatmentEnabled

启用更严格的混合内容处理。

过时的策略

● ForceNetworkInProcess

强制网络代码在浏览器进程中运行。
