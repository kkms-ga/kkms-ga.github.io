Windows 10功能体验包暗示了这款操作系统未来更加模块化

June 13, 2020

微软在 Windows 10的2020年5月更新的系统设置->关于下，增加了一个名为 “Windows功能体验包 “的新项目，适用于Windows 10和服务器上已经将PC更新到2004版本(2020年5月更新)的用户。除了该功能是一个 “关键 “外壳组件的集合，安装大小为44.15 MB，微软并没有给出详细信息，说明该包可能是什么或有什么作用，这让一些人认为它可能是在为Windows 10未来更加模块化做准备，允许该公司在常规操作系统更新之外为核心应用推送功能更新。

Windows 10的2020年5月更新为操作系统带来了几项新功能和变化，并不是所有这些功能和变化都会立即可见，或者看起来对用户有用。其中之一就是随本次更新自动安装的Windows功能体验包，作为微软 “按需提供功能 “的一部分。功能列表中包括几个知名的Windows程序和组件，如记事本、Wordpad、Paint、PowerShell ISE、DirectX配置数据库、OpenSSH客户端和Quick Assistant等，微软将其捆绑在Windows 10镜像文件（ISO）中。

ZDNet的Mary Jo Foley表示，与之前操作系统版本(XP、Vista和7)的Windows体验包影响到Live Messenger和系统主题不同，这个新包可能是微软将功能捆绑在一起的一种方式，更新速度会比Windows 10操作系统本身更快。目前功能体验包中的程序列表相当少，包括Snipping工具、文本输入面板和shell-suggestion UI的更新版本。通过这种模块化的方式，微软现在将能够绕过Windows Update，通过微软商店发布这些组件的更新版本，以及后来添加的组件。这项努力似乎也符合该公司修订后的Windows 10X计划，使其能够通过在未来的Insider构建中捆绑在一起测试外壳组件，并为即将到来的设备优化操作系统的UI/UX。

除了Windows Feature Experience Pack和2004版中的其他一些更改外，更新还包含了常见的bug。根据Windows最新消息，2020年5月的更新导致了微软优化驱动器工具出现问题。尽管该实用程序功能是对存储介质进行碎片整理或者修复，但程序无法报告上次优化磁盘时的正确状态。虽然对于运行第三方程序维护驱动器的用户来说这不是问题，但对于那些仅仅依赖微软工具的用户来说，这可能是一个潜在的问题。该公司在之前的内部版本中已经意识到了这个问题，19551年1月的预览版本中包含了一个修正。然而，微软在2020年5月更新中再次忽略了这个问题，现在这个问题有望在未来的累积更新中得到修补。
