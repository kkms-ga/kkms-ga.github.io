微软发布Edge 85稳定版 但触笔用户可能对Edge 86更感兴趣

August 29, 2020

微软近日向Windows和Mac用户稳定频道发布了Edge 85，新版带来了一些有用的新功能、改进、安全更新、错误修复。那些在PC或Mac上安装了Edge 85的用户现在可以在网络浏览器中打开的PDF文档中突出显示文本，这要归功于一个新的荧光笔工具。

新版Edge还包括对存储访问API的支持，根据Chromestatus的说法，用户可以手动允许在第三方上下文中访问存储器，否则将被浏览器的当前配置所阻止。

此外，现在基于Chromium的Edge 85直接集成了IE浏览器，这意味着用户还可以在浏览器中启用 “IE模式”。换句话说，现在你将能够在Edge浏览器中以Internet Explorer模式启动网站。

QR共享、在Windows启动时运行Edge PWA的能力是其他一些令人兴奋的功能，这些功能在新版Edge上也可以使用，不过要想体验这两个功能，必须分别在设定页面启用名为通过二维码启用共享页面、桌面PWA在操作系统登录时运行的选项。

此版本带来了以下新特性：

收藏夹和设置的本地同步。现在可以在自己的环境中的 Active Directory 配置文件之间同步浏览器收藏夹和设置，而无需云同步。

Microsoft Edge 组策略支持信任站点+应用程序组合启动而无需确认提示。添加了组策略支持，使管理员可以添加可信任的站点+应用程序组合，无需确认提示即可启动。这使管理员能够为其最终用户配置受信任的协议/源组合（例如 Microsoft 365 应用），以在导航到包含应用协议的 URL 时取消确认提示。

PDF 荧光笔工具。可以将该工具添加到 PDF 工具栏中，以轻松高亮重要文本。

Storage Access API 可用。当用户提供直接意图以允许存储时，第三方访问器中的 Storage Access API 允许访问第三方存储，否则将被浏览器的当前配置阻止。更多详情可查看 Storage Access API

Microsoft Edge 集合可发送到 OneNote。可以将收藏夹中的信息发送到 OneNote。更重要的是，在 Microsoft Edge 85 中，用户能够将 MSA 和 Azure Active Directory 的内容发送到 Office for Mac 产品（Word、Excel 和 OneNote）。

DevTools 更新。详情可查看 What’s New In DevTools (Microsoft Edge 85)

支持 Surface Duo 仿真。Microsoft Edge DevTools 现在可以模拟 Surface Duo，因此用户可以测试 Web 内容在双屏设备上的样式。如果要在 DevTools 中打开此实验，请在 Windows上按 Ctrl+Shift+M 或在 macOS 上按 Command+Shift+M 进入“设备模式”，然后从设备下拉列表中选择 Surface Duo。

Microsoft Edge DevTools 可以将键盘快捷键与 VS Code 匹配。Microsoft Edge DevTools 支持在 DevTools 中自定义键盘快捷方式以匹配用户的编辑器/IDE。Microsoft Edge 85 添加了将 DevTools 键盘快捷方式与 VS Code 匹配的功能。此更改将有助于在 VS Code 和 DevTools 之间有效地提高效率。

虽然Edge 85版提供了一些有用的功能，但对于数字笔用户来说，下一个版本的Edge即Edge 86将更加有趣，因为微软宣布将在Edge 86中为PDF的Highlighter工具添加数字笔支持，相关功能已经在Edge 86 Canary和Dev出现。如果想要测试一下，可以从这里获取。
