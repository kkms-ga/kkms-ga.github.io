Windows 10 2004更新为PacketMon网络诊断工具添加了新功能

May 26, 2020

近年来，由于网络徐计划和其它高级功能的加入，调试过程的复杂性已经大大增加。虽然 wireshark 之类的高级数据包捕获工具能够为开发人员提供诊断帮助，但这超出了普通用户的知识范围。但其实早在 2018 年，微软就已经在 Windows 操作系统中内置了网络诊断工具 Packet Monitor（简称 PacketMon）。

访问:

微软中国官方商城 - Windows

（来自：Microsoft）

据悉，PacketMon 提供了数据包捕获、过滤、丢包检测、以及计数等功能，且可在整个网络堆栈的对个位置拦截路由中的数据包。

最近，微软还通过 2020 年 5 月更新（Version 2004），为 Windows 10 和 Windows Server 2019 的 PacketMon 工具添加了新的功能。

● 可在网络堆栈的多个位置捕获数据包；

● 丢包检测，丢包原因报告；

● 支持运行时封装数据包的筛选；

● 提供灵活的数据包计数器；

● 可在屏幕上实施监测数据包；

● 提供高容量的内存日志记录；

● 兼容现有的微软网络监视器（NetMon）和 Wireshark（pcapng）工具。

当然，PacketMon 也有其局限性。因为它只支持在以太网内使用、未与防火墙集成，且丢包报告功能仅适用于受支持的组件。

感兴趣的朋友可运行 pktmon.exe 命令、或者 Windows Server 中的 Windows 管理中心扩展来访问 Packet Monitor（PacketMon）。
