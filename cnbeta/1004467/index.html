微软称Edge已拥抱Fluent设计 短期内无缘透明主题或方块选项卡

July 17, 2020

对于已经看腻了经典版 Edge 浏览器外观的用户来说，换用 Chromium 内核的新版 Edge 浏览器已拥抱更加清新一致的 Fluent 设计风格。Tech Community 的一篇报道称，微软已让新版 Edge 浏览器沿着当前的 Fluent 设计趋势去发展。而早期概念中强调的透明度等元素，现在也已经过时了。

（来自：Microsoft）

微软在文中探讨了大家对新版 Edge 浏览器总体设计的各项反馈，并且围绕 Fluent 设计风格的定义和全局一致性发表了官方看法。

与一组特定或静态的设计规则不同，Fluent Design 旨在随着微软产品的发展而不断进步。随着周围世界的不断发展，人们对于设备和交互模型的使用习惯和期望也会有所变化。

微软希望系统设计也能够持续满足用户不断变化的实际需求，但这也意味着 Fluent 并不总是像它在几年前初次登陆经典版 Edge 浏览器时那样。

感兴趣的朋友可移步至 Fluent Design 的专门介绍页面，以了解微软是如何将之应用于不同的平台、应用程序和控件上的。

翻阅了 Fluent Design 的设计指南之后，你会发现新版 Edge 浏览器已同当前的发展方向保持一致。想必在这之后，该公司的其它产品和服务也会陆续跟进。

Fluent Design 的最新转变，已在对象和控件的圆角上得到体现，此外 Edge 开发团队并没有将重点放在透明度上。

当然，这并不意味着 Edge 永远不会考虑将这些因素纳入其设计的考量范围、或者可能不会提供自定义外观的选项，只是当前尚未考虑将这些纳入面向所有用户的默认体验中。

从路线图来看，Edge 团队将继续与更广泛的 Fluent 设计生态深度融合（比如更新图标和插图），同时欢迎大家继续积极提交有关 Edge 整体外观的反馈（包括偏好和痛点），以便在未来作出改进。
