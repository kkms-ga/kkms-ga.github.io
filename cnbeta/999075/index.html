微软解释为何移除推迟Windows 10 2004功能更新的选项

July 04, 2020

上周，外媒报道了微软从 Windows 10“设置”菜单中悄悄移除了“延迟更新”选项的消息。虽然未向外界正式公布，但这项改动还是被隐藏在了面向 IT 专业人士的文档中。此前，该选项常被用户拿来避免由未经充分验证的更新补丁所带来的一系列问题，直到微软将之调教得足够稳定。然而微软的解释是，为了防止某种程度上的策略混淆，他们还是决定移除这个选项。

微软在近日的公告中称，其已在 Windows 10 Version 2004 中移除了 Windows Update 中的两个高级设置选项。

之后 Windows 10 专业版、企业版、教育版的用户，将无法把特定的功能或质量更新推迟到‘更久’的时间。

软件巨头补充道，由于此选项造成的混淆，导致一些企业客户在无意间更新到新版本。在本周早些时候的博客文章中，Aria Carley 解释称：

除非当前系统即将终止服务，否则用户实际上无需调节任何设置，因为微软不会自动在用户设备上安装新的 Windows 10 功能更新。

将更新推迟 365 天的选项，似乎可以让用户放心一年的时间。但考虑到微软每年都会将提供两次功能更新，某些版本只隔六个月（而不是原定的 365 天）就升级了一次。

只有在没有设置推迟更新的情况下，才有可能停留于当前的版本，直到你选择下载并安全最新的功能更新、或者系统已接近服务终止期。

Carley 解释称，在更改了这项设置之后，你可在任何给定的版本上停留更长的时间（大约每年或更短时间进行一次功能更新），具体取决于用户所使用的 Windows 10 版本。

延期可能是向组织中的一系列设备上部署更新的好方法，但对于最终用户而言，这反而会引起混淆和意外升级、甚至产生某些意想不到的后果。

为防止这些意外后果、并使用户能够在给定的 Windows 10 版本上使用更长的时间 —— 从 Windows 10 Version 2004 开始，微软将不再在 Windows Update 设置页上的“高级”选项卡下显示推迟更新的选项。

这可确保控制和查看何时安装最新的 Windows 10 功能更新，直到您的系统已接近服务终止期。不过延迟功能并未被彻底取消，它只是默认不再显示于设置页面之中。

如有需要，用户仍可通过组策略编辑器（gpedit.msc），在“计算机配置 -> 管理模板 -> Windows 组件 -> Windows Update -> Windows for Business”中设置“何时接收预览版本和功能更新”、以及“何时接收质量更新”。
