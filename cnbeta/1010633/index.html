Linux Mint 官方：LMDE 是具有战略投资意义的 plan B

August 02, 2020

Linux Mint 官方在七月通讯中公布了一项关于各发行版流行度小调查的结果。调查所获信息通过用户默认浏览器起始页的浏览统计量生成，旨在观察各发行版的使用人数。

目前用户占比量最大的依然是 Linux Mint 19.x 系列，超过一半用户都在使用。据官方说明，最新发布的 20.x 系列新增用户增长速度比其他版本都快，但仍未拥有最大用户数量。和 20.x 系列一样，Linux Mint 18.x 系列也占比 20% 左右。

另外，尽管已经结束了生命周期，Linux Mint 17.x 仍有 6％ 的用户在使用。

而基于 Debian 的 LMDE 占比仅 1％。Linux Mint 认为没有关系，毕竟它“没有得到足够推广或获得相同程度的曝光，其目的也不是与 Linux Mint 竞争或吸引新用户”。Linux Mint 官方表示目前将其作为 plan B，称其是一项具有战略意义的投资：“只要 LMDE 的成本不高、质量高，并且我们能够从战略和开发方面吸取经验教训，那它就是一个成功的项目。”

LMDE 在另一个侧面也有助于观察现有版本对 Ubuntu 的依赖程度，观察在没有 Ubuntu 的情况下，有多少工作会被涉及。

将所有发行版进一步细分统计的话，可以发现，每个系列的最后一个发行版（即图表上标有 * 的 “.3” 发行版）在各自的系列中最受欢迎。除了新功能之外，“.3” 发行版代表每个软件包最成熟的版本。

较早的系列和 “.3” 发行版的受欢迎可能意味着，对部分保守的用户群来说，发行版的成熟度比新功能更加重要。

Linux Mint 直呼“我们低估了使用 ’.3‘ 版本的人数”。他们表示，之后在开发新版本时，在向后移植修补程序和解决方案时，会做得更好，并考虑更多。

此外，通讯中也提到了近期用户比较关注的两个问题。一是对 Chromium 的打包问题，此前由于上游系统会直接用 Snap 覆盖 Chromium 的 APT 软件包，Linux Mint 选择不再支持 Snap，并保留 Chromium 的空包，然后指引用户另外的获取方式。但这对用户来说比较麻烦，目前 Linux Mint 还未提供其他解决方式。

另一方面，在升级过程中，用户可能会在处理来自不同国家的软件包时存在困惑及授权问题。Linux Mint 表示目前正着力研究这两个方面，之后会给出解决方案。
