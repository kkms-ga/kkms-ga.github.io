跟谁学回应浑水二度做空报告：不存在机器人刷单

May 29, 2020

5月29日，跟谁学回应浑水二度做空报告，称不存在机器人刷单。跟谁学表示，公司未遗漏时间戳（Timestamp）等关键信息；0.78%IP重合度的具体的计算过程SQL代码已经在第一次回应中展示；每个辅导老师手动切换自己的小班到大班的时间是不相同的，并非所有人都需要同一时间切换至大班，因此并非100%都在同一时间进入大班；

Burst Joiners在开课前后5分钟都有出现是辅导老师手动切换大小班造成的个别情况；公司招聘手机技术工程师是为满足运维需求。

智通财经了解到，5月28日晚间，浑水发布针对跟谁学的第二份做空报告，在新报告中，浑水坚称，跟谁学存在巨大的欺诈行为，几乎是一个空壳。跟谁学对“机器人刷单”的回应是在“混淆视听”，它遗漏了“代理id”、“代理策略”和 “时间戳”三个关键信息，导致其出示的证据“不可用”。另外，浑水出示了跟谁学公开招募工程师帮助其运营机器人的证据——部分跟谁学在招聘网站的招聘信息截图。

以下是跟谁学针对浑水第二份做空报告的回应：

即使仅仅6页，我们仍然坚持进行详细的阅读分析，以及反复的数据校验，很遗憾，浑水对于跟谁学之前的回应也缺乏必要的认知和理解。为了进一步帮助市场了解跟谁学的回应和业务模式，我们进一步做更细节的解释:

  1. 浑水提出 GSX 未在日志记录截屏中展示时间戳(Timestamp)等关键信息

Agent ID and agent policy 为分布式服务器内部节点处理的编号和消息处理规则。GSX 的数据源为直播系统日志，输出的数据结果与上述两个字段无关，且与用户行为无关。我们发布的截屏中，清清楚楚的显示了 timestamp,标记为 ts，该字段“ts:1589716841”的意思如下图所示，时间为2020年5月17日8:00:41PM，我们特地选取浑水报告前一天的日志，方便读者对应。

  1. 浑水提出无法复算得出0.78% IP 重合度，我们以目前数据库中2020年1月至3月的正价课全量数据得出的 IP 地址重合度仅为0.78%。具体的计算过程 SQL 代码已经在第一次回应中展示。

  2. 浑水提出 Precise Joiners 占比为10.6%，比预计低

如我们之前披露的，主讲老师大班开课前，辅导老师会手动将直播间从小班模式切换至主 讲老师主导的大班模式。如果辅导老师没有及时切换，直播系统会自动进行补充切换，切 换的时点一般为课程表中标注的开课时点前后。每个辅导老师手动切换自己的小班到大班 的时间是不相同的，并非所有人都需要同一时间切换至大班，因此并非100%都在同一时 间进入大班。

  1. 浑水提出 Burst Joiners 在开课前后5分钟都有出现

在自动切班系统设置中，如果课程开始前规定时间内大班主讲老师和小班辅导老师未同时 在线，则无法触发自动切换功能，需小班辅导老师手动进行切换，因此在浑水报告中提到 的开课后4分7秒的少量人次，应该是由于以上原因晚开课，从而辅导老师手动切换造成 的个别情况。

5.浑水提出跟谁学公开招聘机器人运维

本公司招聘手机技术工程师和设置手机机房的目的是，为了满足辅导老师团队和销售团队 对手机运维的需求，保证并持续提高我们的教育服务质量和运营效率。跟谁学的业务正在 快速发展中，为了在学生人数不断增加的前提下，跟谁学一直在不断扩大辅导老师队伍， 目前跟谁学已有数千名辅导老师及销售人员，预计到年底，辅导老师的人数将突破1万 人。而辅导和销售工作需要通过手机来完成，这就意味着公司对手机运维的需求也在不断提高。
