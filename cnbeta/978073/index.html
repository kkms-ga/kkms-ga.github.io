[图]用户抱怨Ubuntu 20.04系统无任何snap安装进度提醒

May 12, 2020

数周前Canonical发布了Ubuntu 20.04正式版，引入了诸多重要的改进。不过该版本中出现了一些问题，破坏了用户的使用体验。其中抱怨最多的就是应用商城没有达到用户预期，越来越多的用户在试图安装snaps时遇到各种问题。

其中一个最常见的问题，在外媒SoftPedia编辑的Ubuntu 20.04笔记本上也复现了，涉及安装新snap文件的行为。在点击安装按钮后，没有任何关于安装进度的提示，再次点击同样的按钮会返回一个错误的提示，说明安装过程还在进行中。

正如在Reddit帖子中所推荐的，临时解决方案就是使用Snap Change命令来监控安装进程，不过百分比进度仍然会丢失。

有用户在Reddit社区中写道：

我对snap没有什么意见，但是捆绑的snap store似乎就是完全坏了。我卸载了它，用 sudo apt install gnome-software 取而代之，这似乎少了很多bug，而且还能让你重新获得 flatpak 支持。

要删除snap商店，只需使用以下命令即可：

sudo snap remove snap-store
