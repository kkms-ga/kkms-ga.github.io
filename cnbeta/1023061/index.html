[图]微软承认ESENT错误警告问题 后续更新将修复

September 01, 2020

今年7月，部分用户反馈称升级 Windows 10 May 2020（20H1/Version 2004）功能更新之后，收到了来自 ESENT 的文件警告。现在微软已经承认存在这个问题，并将在未来发布的累积更新中修复这个问题。

在2020年8月21日发布的 Windows 10 Build 20197 更新日志中，微软写道：“根据反馈，我们将关闭 ESENT Warning Event ID 642”。

自 Windows 2000 系统以来的所有版本，都存在可扩展存储引擎（Extensible Storage Engine，缩写ESE）中 ESENT.DLL 文件，被 Windows Update 等诸多 Windows 组件广泛使用。

根据用户的反馈和外媒 Windows Latest 的测试，在事件查看器中可以复现这个 ESENT 642 警告。在事件查看器中的警告如下

Video.UI (23680,D,2) {B8A5865B-DCFF-4019-AA40-BEE2E42C0672}: 由于目前的数据库格式为 1568.20.0，因此无法使用数据库格式功能版本9080 (0x2378)，由参数 0x410022D8 (8920 | JET_efvAllowHigherPersistedFormat)控制。
