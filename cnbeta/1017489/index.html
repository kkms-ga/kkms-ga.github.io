Rust再回应Mozilla裁员：Rust团队成员不受影响，计划创建Rust基金会

August 19, 2020

在 Mozilla 裁员事件闹得沸沸扬扬之际，背后主要是 Mozilla 的 Rust 语言项目团队在发布了一个简短声明，表示 Rust 项目没有危险之后，又紧接着发布了一篇官方博客，针对此事件进行了详细说明。

2020 年 8 月 11 日星期二，Mozilla 宣布他们决定重组公司，并裁员约 250 人，其中包括 Rust 项目和 Rust 社区的活跃成员。可以理解的是，这些裁员对 Rust 项目本身的影响产生了很多不确定性和混乱。我们在这篇文章中的目标是解决这些担忧。同时，我们也有一个重要的消息要宣布。

Rust Core 团队称，尽管产生了深远的个人影响，但 Rust 项目作为一个整体还是对此类事件具有很大的适应性。“我们有来自不同背景和雇主的领导者和贡献者，这种多样性是一种重要的力量。此外，一个普遍的误解是，所有参加 Rust 领导的 Mozilla 员工都把这个当做了他们工作的一部分。实际上，Rust 领导层中的许多 Mozilla 员工都是在个人时间内为 Rust 做出贡献的，而不是作为工作的一部分。”

“最后，我们要强调，Rust 团队的成员资格是授予个人的，并且与雇主无关。即使受到裁员的影响，但原本是 Rust 团队成员的 Mozilla 员工如今仍然是Rust 团队成员。当然，有些人可能会选择减少他们的参与。我们了解并非所有人都能继续做出贡献，我们将完全支持他们的决定。我们感谢他们迄今为止为该项目所做的一切。”

而要宣布的重要消息指的则是，Rust Core 团队和 Mozilla 计划创建一个 Rust 基金会。“去年，我们开始调查创建一个独立的 Rust 基金会的想法。”

Rust Core 团队透露，其目标是在今年年底前完成基金会的第一次迭代。这个基金会的第一个任务将是：掌握所有权。与 Rust、Cargo 和 crates.io 相关的各种商标和域名都将转移到基金会中，基金会也将为它们所产生的费用承担财务责任。目前，Rust Core 团队已经挑选出成员组成一个项目组，推动成立基金会的工作。

“我们认为这个基金会的第一次迭代只是一个开始。基金会的作用还有很多增长的可能性，我们很高兴在未来探索这些。”

此外，Rust Core 团队还表示，在过去两年里，团队一直在致力于减少对任何单一公司赞助项目的依赖，以及增加支持 Rust 的公司数量。现如今，Rust 已经得到了整个生态系统中许多不同公司的支持。“随着我们从法律上过渡为完全独立的实体，基础架构团队计划继续努力，以确保我们不会过分依赖任何一个赞助商。”
