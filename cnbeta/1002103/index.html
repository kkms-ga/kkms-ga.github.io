[图]轻装上阵：WCOS将为Chromium提供现代游戏手柄支持

July 12, 2020

Windows 10 系统的主要优势之一就是，对旧款应用程序和硬件的向后兼容性支持。固然，这个优点让 Windows 新系统支持过去十年间创建的绝大多数应用程序和游戏，但也让系统变得更加臃肿，导致很多遗留内容无法精简。

尽管传闻中的 Andromeda 移动操作系统已经胎死腹中，但相关的模块化系统 Windows Core OS（简称WCOS）目前仍在稳步推进中。WCOS 将会带来脱胎换骨的全新体验，是为 Surface Neo、ChromeBook 等设备的新形态而准备的，可以说是 Windows 系统的未来。

目前，WCOS 使用的是“Composable Shell”，即现有Windows Shell的模块化版本。它能够轻松创建诸如 Windows 10X 这样的新版本系统，在不同的设备外形之间共享功能。

此前就有消息称，微软已经为 Windows Core OS 优化 Chromium。现在出现了更多的证据。微软正在开发一项新功能，该功能将允许 Chromium（Chrome、Edge和其他浏览器）在 Windows 10 上支持更广泛的现代游戏手柄设备。

这项改变将在 Chromium 上默认启用“Windows.Gaming.Input (WGI)”，这应该允许WGI为游戏手柄特定的Plug & Play API提供支持，甚至在 WCOS 设备上。微软在一份支持文件中指出，WGI 为 Windows 应用和游戏提供了对游戏手柄输入的访问。

该技术巨头认为，WGI的支持也可以让 Chromium 支持 Xbox One游戏手柄上的摇杆和按钮，以及在 Windows 10、Windows Core OS(WCOS) 和Windows 10X 上使用基于 Chromium 的浏览器时，支持 Elite 手柄上的拨片。

微软指出：“虽然像Raw Input这样的低级API不支持WCOS设备，但通过这个WGI Gamepad JavaScript API，Chromium获得了对游戏手柄专用Plug & Play API的支持，甚至在WCOS设备上。这个功能可以在Windows桌面（Win32应用）和Windows 10X上都能使用。”
