Windows 10六月累积更新导致系统错误阻止应用程序的运行

June 13, 2020

在 2020 年 6 月的 Windows 10 累积更新发布后，一些用户汇报称其遭遇了打印等问题，甚至在启动 Office 等应用程序时收到“系统找不到某 .exe 可执行文件”的报错信息。从大家的反馈来看，该问题似乎与 Windows 10 1909 的 KB4560960、以及 Windows 10 2004 的 KB4557957 累积更新有关。

需要指出的是，Avast 于近日发布了一个更新，导致用户会遇到一个类似的阻止应用程序运行的报错信息：

Windows 无法找到 C:Program Files (x86)Microsoft OfficerootOffice16EXCEL.EXE，请确保您已输入正确的名称，然后再次尝试。

然而用户指出，即便他们没有使用或安装过 Avast 反病毒软件，系统仍然会弹出这样的报错信息。

不过还有一些报告称，Windows 10 累积更新和 Avast 更新都会导致这些应用程序的功能被破坏，导致其无法正常运行。

之后微软社区志愿者证实这是 Windows 10 本次累积更新带来的一个“已知问题”，且用户可以通过调整注册表项目来修复：

● 打开注册表编辑器（运行 regedit）；

● 转至 ComputerHKEY_LOCAL_MACHINESOFTWAREMicrosoftWindows NTCurrentVersionImage File Execution 路径；

● 对无法顺利运行的应用程序文件夹进行重命名（比如将 Chrome.exe 改为 BackupChrome.exe）；

● 然后尝试重新启动计算机。

如果在执行了上述操作后（且未使用 Avast 反病毒软件），问题依然存在，你可尝试卸载 KB4560960 或 KB4557957 这两个 Windows 10 累积更新（分别对应 1909 / 2004 的系统版本）

最后 Avast 公司建议自家反病毒软件用户升级到最新版本，以避免遇到类似的问题。
